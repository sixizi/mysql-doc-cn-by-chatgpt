#### 25.4.2.5 NDB Cluster `mysqld` 选项和变量参考

以下列表包括在 `mysqld` 作为 NDB Cluster 中的 SQL 节点运行时适用的命令行选项、系统变量和状态变量。有关与 `mysqld` 相关的所有命令行选项、系统变量和状态变量的参考，请参见 [第 7.1.4 节 “服务器选项、系统变量和状态变量参考”](#7.1.4-server-option-system-variable-and-status-variable-reference)。

- `Com_show_ndb_status`：`SHOW NDB STATUS` 语句的计数。
- `Handler_discover`：发现表的次数。
- `ndb-applier-allow-skip-epoch`：允许复制应用程序跳过纪元。
- `ndb-batch-size`：用于 NDB 事务批处理的大小（字节）。
- `ndb-blob-read-batch-bytes`：指定大 BLOB 读取应批处理的大小（字节）。0 = 无限制。
- `ndb-blob-write-batch-bytes`：指定大 BLOB 写入应批处理的大小（字节）。0 = 无限制。
- `ndb-cluster-connection-pool`：MySQL 使用的集群连接数。
- `ndb-cluster-connection-pool-nodeids`：MySQL 使用的连接到集群的节点 ID 列表，用逗号分隔；列表中的节点数量必须与 `--ndb-cluster-connection-pool` 设置的值匹配。
- `ndb-connectstring`：分发此集群配置信息的 NDB 管理服务器地址。
- `ndb-default-column-format`：创建或添加表列时默认使用的 `COLUMN_FORMAT` 和 `ROW_FORMAT` 值（`FIXED` 或 `DYNAMIC`）。
- `ndb-deferred-constraints`：指定对唯一索引的约束检查（如果支持）应延迟到提交时。通常不需要或不使用；仅用于测试目的。
- `ndb-distribution`：NDBCLUSTER 中新表的默认分布方式（`KEYHASH` 或 `LINHASH`，默认为 `KEYHASH`）。
- `ndb-log-apply-status`：使作为副本的 MySQL 服务器将从其直接源接收到的 `mysql.ndb_apply_status` 更新记录到其自己的二进制日志中，仅在服务器以 `--ndbcluster` 选项启动时有效。
- `ndb-log-empty-epochs`：启用后，即使启用了 `--log-slave-updates`，在没有更改的情况下，将纪元写入 `ndb_apply_status` 和 `ndb_binlog_index` 表。
- `ndb-log-empty-update`：启用后，即使启用了 `--log-slave-updates`，在没有产生更改的情况下，将更新写入 `ndb_apply_status` 和 `ndb_binlog_index` 表。
- `ndb-log-exclusive-reads`：记录具有排他锁的主键读取；允许基于读取冲突的冲突解决。
- `ndb-log-fail-terminate`：如果无法完整记录所有找到的行事件，则终止 `mysqld` 进程。
- `ndb-log-orig`：在 `mysql.ndb_binlog_index` 表中记录原始服务器的 ID 和纪元。
- `ndb-log-transaction-dependency`：使二进制日志线程计算写入二进制日志的每个事务的事务依赖性。
- `ndb-log-transaction-id`：在二进制日志中写入 NDB 事务 ID。需要 `--log-bin-v1-events=OFF`。
- `ndb-log-update-minimal`：以最小格式记录更新。
- `ndb-log-updated-only`：仅记录更新（`ON`）或完整行（`OFF`）。
- `ndb-log-update-as-write`：在源上记录更新（`OFF`）和写入（`ON`）之间切换。
- `ndb-mgmd-host`：设置连接到管理服务器的主机（和端口，如果需要）。
- `ndb-nodeid`：此 MySQL 服务器的 NDB Cluster 节点 ID。
- `ndb-optimized-node-selection`：启用选择事务节点优化。默认启用；使用 `--skip-ndb-optimized-node-selection` 禁用。
- `ndb-transid-mysql-connection-map`：启用或禁用 `ndb_transid_mysql_connection_map` 插件，即启用或禁用具有该名称的 `INFORMATION_SCHEMA` 表。
- `ndb-wait-connected`：MySQL 服务器在接受 MySQL 客户端连接前等待连接到集群管理和数据节点的时间（秒）。
- `ndb-wait-setup`：MySQL 服务器等待 NDB 引擎设置完成的时间（秒）。
- `ndb-allow-copying-alter-table`：设置为 `OFF`，以防止 `ALTER TABLE` 在 NDB 表上使用复制操作。
- `Ndb_api_adaptive_send_deferred_count`：此 MySQL 服务器（SQL 节点）未实际发送的自适应发送调用数。
- `Ndb_api_adaptive_send_deferred_count_session`：此客户端会话中未实际发送的自适应发送调用数。
- `Ndb_api_adaptive_send_deferred_count_replica`：此副本未实际发送的自适应发送调用数。
- `Ndb_api_adaptive_send_deferred_count_slave`：此副本未实际发送的自适应发送调用数。
- `Ndb_api_adaptive_send_forced_count`：此 MySQL 服务器（SQL 节点）发送的具有强制发送设置的自适应发送数。
- `Ndb_api_adaptive_send_forced_count_session`：此客户端会话中发送的具有强制发送设置的自适应发送数。
- `Ndb_api_adaptive_send_forced_count_replica`：此副本发送的具有强制发送设置的自适应发送数。
- `Ndb_api_adaptive_send_forced_count_slave`：此副本发送的具有强制发送设置的自适应发送数。
- `Ndb_api_adaptive_send_unforced_count`：此 MySQL 服务器（SQL 节点）发送的无强制发送的自适应发送数。
- `Ndb_api_adaptive_send_unforced_count_session`：此客户端会话中发送的无强制发送的自适应发送数。
- `Ndb_api_adaptive_send_unforced_count_replica`：此副本发送的无强制发送的自适应发送数。
- `Ndb_api_adaptive_send_unforced_count_slave`：此副本发送的无强制发送的自适应发送数。
- `Ndb_api_bytes_received_count`：此 MySQL 服务器（SQL 节点）从数据节点接收的数据量（字节）。
- `Ndb_api_bytes_received_count_session`：此客户端会话中从数据节点接收的数据量（字节）。
- `Ndb_api_bytes_received_count_replica`：此副本从数据节点接收的数据量（字节）。
- `Ndb_api_bytes_received_count_slave`：此副本从数据节点接收的数据量（字节）。
- `Ndb_api_bytes_sent_count`：此 MySQL 服务器（SQL 节点）发送到数据节点的数据量（字节）。
- `Ndb_api_bytes_sent_count_session`：此客户端会话中发送到数据节点的数据量（字节）。
- `Ndb_api_bytes_sent_count_replica`：此副本发送到数据节点的数据量（字节）。
- `Ndb_api_bytes_sent_count_slave`：此副本发送到数据节点的数据量（字节）。
- `Ndb_api_event_bytes_count`：此 MySQL 服务器（SQL 节点）收到的事件字节数。
- `Ndb_api_event_bytes_count_injector`：NDB 二进制日志注入线程收到的事件数据字节数。
- `Ndb_api_event_data_count`：此 MySQL 服务器（SQL 节点）收到的行更改事件数。
- `Ndb_api_event_data_count_injector`：NDB 二进制日志注入线程收到的行更改事件数。
- `Ndb_api_event_nondata_count`：此 MySQL 服务器（SQL 节点）收到的非行更改事件数。
- `Ndb_api_event_nondata_count_injector`：NDB 二进制日志注入线程收到的非行更改事件数。
- `Ndb_api_pk_op_count`：此 MySQL 服务器（SQL 节点）基于或使用主键的操作数。
- `Ndb_api_pk_op_count_session`：此客户端会话中基于或使用主键的操作数。
- `Ndb_api_pk_op_count_replica`：此副本基于或使用主键的操作数。
- `Ndb_api_pk_op_count_slave`：此副本基于或使用主键的操作数。
- `Ndb_api_pruned_scan_count`：此 MySQL 服务器（SQL 节点）已修剪到一个分区的扫描数。

- `Ndb_api_pruned_scan_count_session`：此客户端会话中已修剪到一个分区的扫描数。
- `Ndb_api_pruned_scan_count_replica`：此副本已修剪到一个分区的扫描数。
- `Ndb_api_pruned_scan_count_slave`：此副本已修剪到一个分区的扫描数。
- `Ndb_api_range_scan_count`：此 MySQL 服务器（SQL 节点）启动的范围扫描数。
- `Ndb_api_range_scan_count_session`：此客户端会话中启动的范围扫描数。
- `Ndb_api_range_scan_count_replica`：此副本启动的范围扫描数。
- `Ndb_api_range_scan_count_slave`：此副本启动的范围扫描数。
- `Ndb_api_read_row_count`：此 MySQL 服务器（SQL 节点）读取的总行数。
- `Ndb_api_read_row_count_session`：此客户端会话中读取的总行数。
- `Ndb_api_read_row_count_replica`：此副本读取的总行数。
- `Ndb_api_read_row_count_slave`：此副本读取的总行数。
- `Ndb_api_scan_batch_count`：此 MySQL 服务器（SQL 节点）收到的行批次数。
- `Ndb_api_scan_batch_count_session`：此客户端会话中收到的行批次数。
- `Ndb_api_scan_batch_count_replica`：此副本收到的行批次数。
- `Ndb_api_scan_batch_count_slave`：此副本收到的行批次数。
- `Ndb_api_table_scan_count`：此 MySQL 服务器（SQL 节点）启动的表扫描数，包括内部表的扫描。
- `Ndb_api_table_scan_count_session`：此客户端会话中启动的表扫描数，包括内部表的扫描。
- `Ndb_api_table_scan_count_replica`：此副本启动的表扫描数，包括内部表的扫描。
- `Ndb_api_table_scan_count_slave`：此副本启动的表扫描数，包括内部表的扫描。
- `Ndb_api_trans_abort_count`：此 MySQL 服务器（SQL 节点）中止的事务数。
- `Ndb_api_trans_abort_count_session`：此客户端会话中中止的事务数。
- `Ndb_api_trans_abort_count_replica`：此副本中止的事务数。
- `Ndb_api_trans_abort_count_slave`：此副本中止的事务数。
- `Ndb_api_trans_close_count`：此 MySQL 服务器（SQL 节点）关闭的事务数；可能大于 `TransCommitCount` 和 `TransAbortCount` 的总和。
- `Ndb_api_trans_close_count_session`：此客户端会话中关闭的事务数（可能大于 `TransCommitCount` 和 `TransAbortCount` 的总和）。
- `Ndb_api_trans_close_count_replica`：此副本关闭的事务数（可能大于 `TransCommitCount` 和 `TransAbortCount` 的总和）。
- `Ndb_api_trans_close_count_slave`：此副本关闭的事务数（可能大于 `TransCommitCount` 和 `TransAbortCount` 的总和）。
- `Ndb_api_trans_commit_count`：此 MySQL 服务器（SQL 节点）提交的事务数。
- `Ndb_api_trans_commit_count_session`：此客户端会话中提交的事务数。
- `Ndb_api_trans_commit_count_replica`：此副本提交的事务数。
- `Ndb_api_trans_commit_count_slave`：此副本提交的事务数。
- `Ndb_api_trans_local_read_row_count`：此 MySQL 服务器（SQL 节点）读取的总行数。
- `Ndb_api_trans_local_read_row_count_session`：此客户端会话中读取的总行数。
- `Ndb_api_trans_local_read_row_count_replica`：此副本读取的总行数。
- `Ndb_api_trans_local_read_row_count_slave`：此副本读取的总行数。
- `Ndb_api_trans_start_count`：此 MySQL 服务器（SQL 节点）启动的事务数。
- `Ndb_api_trans_start_count_session`：此客户端会话中启动的事务数。
- `Ndb_api_trans_start_count_replica`：此副本启动的事务数。
- `Ndb_api_trans_start_count_slave`：此副本启动的事务数。
- `Ndb_api_uk_op_count`：此 MySQL 服务器（SQL 节点）基于或使用唯一键的操作数。
- `Ndb_api_uk_op_count_session`：此客户端会话中基于或使用唯一键的操作数。
- `Ndb_api_uk_op_count_replica`：此副本基于或使用唯一键的操作数。
- `Ndb_api_uk_op_count_slave`：此副本基于或使用唯一键的操作数。
- `Ndb_api_wait_exec_complete_count`：此 MySQL 服务器（SQL 节点）等待操作执行完成时线程被阻塞的次数。
- `Ndb_api_wait_exec_complete_count_session`：此客户端会话中等待操作执行完成时线程被阻塞的次数。
- `Ndb_api_wait_exec_complete_count_replica`：此副本等待操作执行完成时线程被阻塞的次数。
- `Ndb_api_wait_exec_complete_count_slave`：此副本等待操作执行完成时线程被阻塞的次数。

- `Ndb_api_wait_meta_request_count_session`：此客户端会话中等待元数据信号时线程被阻塞的次数。
- `Ndb_api_wait_meta_request_count_replica`：此副本等待元数据信号时线程被阻塞的次数。
- `Ndb_api_wait_meta_request_count_slave`：此副本等待元数据信号时线程被阻塞的次数。
- `Ndb_api_wait_nanos_count`：此 MySQL 服务器（SQL 节点）等待数据节点某种类型信号的总时间（纳秒）。
- `Ndb_api_wait_nanos_count_session`：此客户端会话中等待数据节点某种类型信号的总时间（纳秒）。

- `Ndb_api_wait_nanos_count_slave`：此副本等待数据节点某种类型信号的总时间（纳秒）。
- `Ndb_api_wait_scan_result_count`：此 MySQL 服务器（SQL 节点）等待基于扫描信号时线程被阻塞的次数。
- `Ndb_api_wait_scan_result_count_session`：此客户端会话中等待基于扫描信号时线程被阻塞的次数。
- `Ndb_api_wait_scan_result_count_replica`：此副本等待基于扫描信号时线程被阻塞的次数。
- `Ndb_api_wait_scan_result_count_slave`：此副本等待基于扫描信号时线程被阻塞的次数。
- `ndb_autoincrement_prefetch_sz`：NDB 自动增量预取大小。
- `ndb_clear_apply_status`：导致 `RESET SLAVE`/`RESET REPLICA` 清除 `ndb_apply_status` 表中的所有行；默认开启。
- `Ndb_cluster_node_id`：此服务器作为 NDB Cluster SQL 节点时的节点 ID。
- `Ndb_config_from_host`：NDB Cluster 管理服务器主机名或 IP 地址。
- `Ndb_config_from_port`：连接到 NDB Cluster 管理服务器的端口。
- `Ndb_config_generation`：集群当前配置的生成编号。
- `Ndb_conflict_fn_epoch`：NDB 复制冲突检测函数 `NDB$EPOCH()` 发现的冲突行数。
- `Ndb_conflict_fn_epoch2`：NDB 复制冲突检测函数 `NDB$EPOCH2()` 发现的冲突行数。
- `Ndb_conflict_fn_epoch2_trans`：NDB 复制冲突检测函数 `NDB$EPOCH2_TRANS()` 发现的冲突行数。
- `Ndb_conflict_fn_epoch_trans`：NDB 冲突检测函数 `NDB$EPOCH_TRANS()` 发现的冲突行数。
- `Ndb_conflict_fn_max`：基于“较大时间戳优先”应用于更新和删除操作的 NDB 复制冲突解决的次数。
- `Ndb_conflict_fn_max_del_win`：基于 `NDB$MAX_DELETE_WIN()` 结果应用于更新和删除操作的 NDB 复制冲突解决的次数。
- `Ndb_conflict_fn_max_ins`：基于“较大时间戳优先”应用于插入操作的 NDB 复制冲突解决的次数。
- `Ndb_conflict_fn_max_del_win_ins`：基于 `NDB$MAX_DEL_WIN_INS()` 结果应用于插入操作的 NDB 复制冲突解决的次数。
- `Ndb_conflict_fn_old`：应用于 NDB 复制“相同时间戳优先”冲突解决的次数。
- `Ndb_conflict_last_conflict_epoch`：此副本上最近检测到冲突的 NDB 纪元。
- `Ndb_conflict_last_stable_epoch`：包含无冲突的最近纪元。
- `Ndb_conflict_reflected_op_discard_count`：由于执行错误未应用的反射操作数。
- `Ndb_conflict_reflected_op_prepare_count`：已准备执行的反射操作数。
- `Ndb_conflict_refresh_op_count`：已准备的刷新操作数。
- `ndb_conflict_role`：副本在冲突检测和解决中扮演的角色。值为 `PRIMARY`、`SECONDARY`、`PASS` 或 `NONE`（默认）。仅在停止复制 SQL 线程时更改。详见文档。
- `Ndb_conflict_trans_conflict_commit_count`：在需要事务冲突处理后提交的纪元事务数。
- `Ndb_conflict_trans_detect_iter_count`：提交纪元事务所需的内部迭代次数。应略大于或等于 `Ndb_conflict_trans_conflict_commit_count`。
- `Ndb_conflict_trans_reject_count`：由事务冲突函数发现冲突后被拒绝的事务数。
- `Ndb_conflict_trans_row_conflict_count`：由事务冲突函数发现的冲突行数。包括任何依赖于冲突事务的行。
- `Ndb_conflict_trans_row_reject_count`：由事务冲突函数发现冲突后重新对齐的总行数。包括 `Ndb_conflict_trans_row_conflict_count` 和任何依赖于冲突事务的行。
- `ndb_data_node_neighbour`：指定此 MySQL 服务器的“最近”集群数据节点，用于事务提示和完全复制的表。
- `ndb_default_column_format`：设置新 NDB 表的默认行格式和列格式（`FIXED` 或 `DYNAMIC`）。
- `ndb_deferred_constraints`：指定应延迟（如果支持）约束检查。通常不需要或不使用；仅用于测试目的。
- `ndb_dbg_check_shares`：检查是否有任何剩余的共享（仅限调试版本）。
- `ndb-schema-dist-timeout`：架构分布期间检测超时前等待的时间。
- `ndb_distribution`：NDBCLUSTER 中新表的默认分布方式（`KEYHASH` 或 `LINHASH`，默认为 `KEYHASH`）。
- `Ndb_epoch_delete_delete_count`：检测到的删除-删除冲突数（删除操作已应用，但行不存在）。
- `ndb_eventbuffer_free_percent`：事件缓冲区达到 `ndb_eventbuffer_max_alloc` 设置的限制后，缓冲事件之前可用的事件缓冲区内存百分比。
- `ndb_eventbuffer_max_alloc`：NDB API 用于缓冲事件的最大内存。默认为 0（无上限）。
- `Ndb_execute_count`：NDB 内核操作的往返次数。
- `ndb_extra_logging`：控制 NDB Cluster 架构、连接和数据分布事件的 MySQL 错误日志记录。
- `ndb_force_send`：强制立即将缓冲区发送到 NDB，而不等待其他线程。
- `ndb_fully_replicated`：新 NDB 表是否完全复制。
- `ndb_index_stat_enable`：在查询优化中使用 NDB 索引统计信息。
- `ndb_index_stat_option`：NDB 索引统计信息的可调选项列表；列表中不应包含空格。
- `ndb_join_pushdown`：启用将连接下推到数据节点。
- `Ndb_last_commit_epoch_server`：NDB 最近提交的纪元。
- `Ndb_last_commit_epoch_session`：此 NDB 客户端最近提交的纪元。
- `ndb_log_apply_status`：是否将作为副本的 MySQL 服务器接收到的 `mysql.ndb_apply_status` 更新记录到其自己的二进制日志中。
- `ndb_log_bin`：将更新写入 NDB 表的二进制日志中。仅在启用了二进制日志记录时有效（使用 `--log-bin`）。
- `ndb_log_binlog_index`：将纪元和二进制日志位置之间的映射插入到 `ndb_binlog_index` 表中。默认开启。仅在启用了二进制日志记录时有效。
- `ndb_log_empty_epochs`：启用后，即使启用了 `log_replica_updates` 或 `log_slave_updates`，在没有更改的情况下，将纪元写入 `ndb_apply_status` 和 `ndb_binlog_index` 表。
- `ndb_log_empty_update`：启用后，即使启用了 `log_replica_updates` 或 `log_slave_updates`，在没有产生更改的情况下，将更新写入 `ndb_apply_status` 和 `ndb_binlog_index` 表。
- `ndb_log_exclusive_reads`：记录具有排他锁的主键读取；允许基于读取冲突的冲突解决。
- `ndb_log_orig`：是否在 `mysql.ndb_binlog_index` 表中记录原始服务器的 ID 和纪元。使用 `--ndb-log-orig` 选项启动 `mysqld` 时设置。
- `ndb_log_transaction_id`：是否将 NDB 事务 ID 写入二进制日志（只读）。
- `ndb_log_transaction_compression`：是否压缩 NDB 二进制日志；也可以通过启用 `--binlog-transaction-compression` 选项在启动时启用。
- `ndb_log_transaction_compression_level_zstd`：将压缩事务写入 NDB 二进制日志时使用的 ZSTD 压缩级别。
- `ndb_metadata_check`：启用 NDB 元数据与 MySQL 数据字典之间的元数据变化自动检测；默认启用。
- `Ndb_metadata_blacklist_size`：NDB 二进制日志线程未能同步的 NDB 元数据对象数；在 NDB 8.0.22 中重命名为 `Ndb_metadata_excluded_count`。
- `ndb_metadata_check_interval`：检查 NDB 元数据与 MySQL 数据字典之间元数据变化的时间间隔（秒）。
- `Ndb_metadata_detected_count`：NDB 元数据更改监视线程检测到的变化次数。
- `Ndb_metadata_excluded_count`：NDB 二进制日志线程未能同步的 NDB 元数据对象数。
- `ndb_metadata_sync`：触发 NDB 字典和 MySQL 数据字典之间所有更改的立即同步；导致忽略 `ndb_metadata_check` 和 `ndb_metadata_check_interval` 值。同步完成后重置为 `false`。
- `Ndb_metadata_synced_count`：已同步的 NDB 元数据对象数。
- `Ndb_number_of_data_nodes`：此 NDB 集群中的数据节点数；仅在服务器参与集群时设置。
- `ndb-optimization-delay`：在 NDB 表上的 `OPTIMIZE TABLE` 操作之间处理行集的等待时间（毫秒）。
- `ndb_optimized_node_selection`：SQL 节点选择集群数据节点作为事务协调器的方式。
- `Ndb_pruned_scan_count`：集群上次启动以来使用分区修剪的扫描数。
- `Ndb_pushed_queries_defined`：API 节点尝试下推到数据节点的连接数。
- `Ndb_pushed_queries_dropped`：API 节点尝试下推但失败的连接数。
- `Ndb_pushed_queries_executed`：成功下推并在数据节点上执行的连接数。
- `Ndb_pushed_reads`：由下推连接在数据节点上执行的读取数。
- `ndb_read_backup`：启用对所有 NDB 表的任何副本读取；使用 `NDB_TABLE=READ_BACKUP={0|1}` 和 `CREATE TABLE` 或 `ALTER TABLE` 来为单个 NDB 表启用或禁用。
- `ndb_recv_thread_activation_threshold`：当接收线程接管集群连接轮询时的激活阈值（以并发活动线程数衡量）。
- `ndb_recv_thread_cpu_mask`：用于将接收线程锁定到特定 CPU 的 CPU 掩码；以十六进制表示。详见文档。
- `Ndb_replica_max_replicated_epoch`：此副本上最近提交的 NDB 纪元。当此值大于或等于 `Ndb_conflict_last_conflict_epoch` 时，尚未检测到冲突。
- `ndb_replica_batch_size`：副本应用程序的批处理大小（字节）。
- `ndb_report_thresh_binlog_epoch_slip`：NDB 7.5 及更高版本：完全缓冲但尚未被二进制日志注入线程消耗的纪元数阈值，超过此阈值会生成 `BUFFERED_EPOCHS_OVER_THRESHOLD` 事件缓冲状态消息；NDB 7.5 之前的版本：报告二进制日志状态之前滞后的纪元数阈值。
- `ndb_report_thresh_binlog_mem_usage`：在报告二进制日志状态之前剩余的自由内存百分比阈值。
- `ndb_row_checksum`：启用时，设置行校验和；默认启用。
- `Ndb_scan_count`：集群上次启动以来执行的总扫描数。
- `ndb_schema_dist_lock_wait_timeout`：架构分布期间等待锁的时间，然后返回错误。
- `ndb_schema_dist_timeout`：架构分布期间检测超时前等待的时间。
- `ndb_schema_dist_upgrade_allowed`：允许架构分布表在连接到 NDB 时升级。
- `ndb_show_foreign_key_mock_tables`：显示用于支持 `foreign_key_checks=0` 的模拟表。
- `ndb_slave_conflict_role`：副本在冲突检测和解决中扮演的角色。值为 `PRIMARY`、`SECONDARY`、`PASS` 或 `NONE`（默认）。仅在停止复制 SQL 线程时更改。详见文档。
- `Ndb_slave_max_replicated_epoch`：此副本上最近提交的 NDB 纪元。当此值大于或等于 `Ndb_conflict_last_conflict_epoch` 时，尚未检测到冲突。
- `Ndb_system_name`：配置的集群系统名称；如果服务器未连接到 NDB，则为空。
- `ndb_table_no_logging`：在启用此设置时创建的 NDB 表不会检查点到磁盘（尽管会创建表架构文件）。表使用或更改为使用 `NDBCLUSTER` 时，此设置有效并持续。
- `ndb_table_temporary`：NDB 表在磁盘上不持久化：不创建架构文件，表也不记录。
- `Ndb_trans_hint_count_session`：此会话中使用提示启动的事务数。
- `ndb_use_copying_alter_table`：在 NDB Cluster 中使用复制的 `ALTER TABLE` 操作。
- `ndb_use_exact_count`：在 SELECT COUNT(*) 查询规划期间强制 NDB 使用记录数以加速此类查询。
- `ndb_use_transactions`：设置为 `OFF`，禁用 NDB 的事务支持。不推荐使用，除非在某些特殊情况下；详见文档。
- `ndb_version`：显示版本和 NDB 引擎版本（整数）。
- `ndb_version_string`：显示包括 NDB 引擎版本在内的版本信息（格式为 `ndb-x.y.z`）。
- `ndbcluster`：启用 NDB Cluster（如果此版本的 MySQL 支持）。通过 `--skip-ndbcluster` 禁用。
- `ndbinfo`：启用 `ndbinfo` 插件（如果支持）。
- `ndbinfo_database`：用于 NDB 信息数据库的名称（只读）。
- `ndbinfo_max_bytes`：仅用于调试。
- `ndbinfo_max_rows`：仅用于调试。
- `ndbinfo_offline`：将 `ndbinfo` 数据库置于离线模式，此模式下表或视图中不返回任何行。
- `ndbinfo_show_hidden`：是否在 MySQL 客户端中显示 `ndbinfo` 内部基表；默认关闭。
- `ndbinfo_table_prefix`：用于命名 `ndbinfo` 内部基表的前缀；只读。
- `ndbinfo_version`：`ndbinfo` 引擎版本；只读。
- `replica_allow_batching`：打开和关闭副本的更新批处理。
- `server_622424198204195214_bits`：在 `server_622424198204195214` 中实际用于标识服务器的最低有效位数，允许 NDB API 应用程序在最高有效位中存储应用程序数据。`server_622424198204195214` 必须小于此值的 2 的幂。
- `skip-ndbcluster`：禁用 NDB Cluster 存储引擎。
- `slave_allow_batching`：打开和关闭副本的更新批处理。
- `transaction_allow_batching`：允许在一个事务中批处理语句。禁用 `AUTOCOMMIT` 以使用。
